# Fraud Guardian - Requirements Document

## Problem Statement
Financial frauds such as phishing, QR scams, OTP scams, and deepfake calls are increasing rapidly. Users lack real-time tools to detect and understand scams.

## Functional Requirements
- Accept text input (SMS, email, WhatsApp messages)
- Accept audio input (call recordings)
- Accept image input (QR codes, screenshots)
- Detect scam probability
- Generate explanation in simple language
- Provide actionable safety recommendations

## Non-Functional Requirements
- Fast response time (<3 seconds)
- High precision in scam detection
- Scalable cloud architecture
- Secure data handling

## Target Users
- UPI users
- Senior citizens
- Small business owners
- Students
